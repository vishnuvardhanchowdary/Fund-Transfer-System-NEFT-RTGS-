# Fund Transfer System (NEFT / RTGS / IMPS)

A complete banking transfer project with beneficiary management, transfer limits, mock OTP verification, status tracking, transaction history, PDF receipts, notifications, admin controls, and REST APIs.

## Highlights

- Beneficiary add, list, validate, and delete
- NEFT, RTGS, and IMPS transfer form
- Transfer limit display with daily usage calculation
- Mock OTP generation and verification
- Status tracker with transaction timeline
- Receipt PDF download endpoint
- Transaction history with CSV export
- Notifications center with read status
- Admin panel for transfer limits and transaction status
- Postman collection for API testing
- Local JSON persistence, no database setup required

## Tech Stack

- HTML5, CSS3, JavaScript ES6+
- Node.js REST API
- Browser `fetch` for API calls
- LocalStorage for transfer draft and UI state
- JSON file storage for demo data
- Built-in PDF generation for receipts

The project is intentionally dependency-free so it can run immediately on Windows with Node.js and npm.

## Run

```bash
npm start
```

Then open:

```text
http://localhost:3000
```

If port `3000` is busy:

```bash
$env:PORT=3100; npm start
```

## Demo Flow

1. Open the Transfer module.
2. Select a saved beneficiary, mode, amount, and remarks.
3. Initiate transfer.
4. Copy the displayed mock OTP.
5. Verify OTP.
6. Open Status Tracker to monitor the transaction.
7. Download the PDF receipt from Status Tracker or History.

## API Endpoints

| Method | Endpoint | Purpose |
| --- | --- | --- |
| GET | `/api/health` | Service health check |
| GET | `/api/dashboard` | Full dashboard data |
| GET | `/api/limits` | Transfer limits and usage |
| PATCH | `/api/limits/:method` | Admin limit update |
| GET | `/api/beneficiaries` | List beneficiaries |
| POST | `/api/beneficiaries` | Add beneficiary |
| PATCH | `/api/beneficiaries/:id` | Update beneficiary |
| DELETE | `/api/beneficiaries/:id` | Delete beneficiary |
| POST | `/api/transfers/initiate` | Create transfer and mock OTP |
| POST | `/api/transfers/:id/verify-otp` | Verify OTP |
| GET | `/api/transactions` | List or search transactions |
| GET | `/api/transactions.csv` | Download transaction CSV |
| GET | `/api/transactions/:id` | Get transaction and refresh status |
| PATCH | `/api/transactions/:id/status` | Admin status update |
| GET | `/api/transactions/:id/receipt.pdf` | Download receipt PDF |
| GET | `/api/notifications` | List notifications |
| PATCH | `/api/notifications/read-all` | Mark notifications read |
| GET | `/api/admin/audit` | Audit log |

## Notes

- Data is stored in `data/store.json`.
- RTGS has a minimum transfer amount of INR 2,00,000.
- Mock OTP is returned by the API and displayed in the UI for demo testing.
- The PDF receipt is generated by the Node server without external libraries.
